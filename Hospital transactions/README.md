# Hospital Transactions

## Project Overview

The Hospital Transactions Dashboard is a Power BI analytics solution designed to provide healthcare stakeholders with a comprehensive view of patient visits, pharmacy transactions, hospital departments, diagnoses, and revenue performance.

The dashboard combines patient information, visit records, and pharmacy transaction data to support operational monitoring, financial analysis, and healthcare decision-making. By consolidating these datasets into an interactive reporting environment, users can quickly identify trends, evaluate service utilization, and assess pharmacy revenue performance.

## Business Problem

Healthcare facilities generate large volumes of transactional and operational data every day. Without proper visualization and analysis tools, it becomes difficult to:

Understand patient visitation trends

Monitor pharmacy revenue performance

Identify frequently occurring diagnoses

Evaluate departmental workloads

Track patient distribution across different counties

Measure patient length of stay and related operational impacts

This dashboard addresses these challenges by transforming raw hospital data into meaningful insights through interactive visualizations and filtering capabilities.

## Dataset Structure

The dashboard is built using multiple related tables that represent different aspects of hospital operations.

### Dim_Patients

Contains patient related information including:

Patient demographics

County information

Calculated healthcare metrics

Visit summaries

### Facts Visits

Contains hospital visit records including:

Visit identifiers

Visit dates

Diagnosis information

Department information

Length of stay details

### Facts Pharmacy Transactions

Contains pharmacy transaction records including:

Transaction costs

Medication purchases

Visit references

Revenue related metrics

### Dim Transactions

Contains pharmacy transaction categorization information including:

Transaction identifiers

Drug categories

Supporting transaction dimensions

## Dashboard Pages

### Page 1: Executive Dashboard

This page provides a high level overview of hospital performance.

#### Key Performance Indicators

Total Visits

Displays the total number of patient visits recorded within the selected period.

Pharmacy Revenue

Shows total revenue generated from pharmacy transactions.

Average Length of Stay

Measures the average number of days patients remain admitted.

#### Visualizations

Monthly Visit Trends

A line chart displaying visit volumes across different months to identify growth patterns and seasonal fluctuations.

Pharmacy Revenue by Drug Category

A column chart showing revenue contribution from various drug categories.

Patient Distribution by County

A funnel chart illustrating patient volumes across counties.

Department Utilization

A donut chart displaying visit distribution across hospital departments.

#### Interactive Filters

County Filter

Allows users to focus analysis on specific geographic regions.

Diagnosis Filter

Enables filtering by medical condition or diagnosis.

Visit Date Filter

Provides time based analysis capabilities.

### Page 2: Detailed Analysis Dashboard

This page focuses on deeper operational and clinical insights.

#### Diagnosis Analysis by County

A clustered column chart comparing diagnosis frequency across counties.

#### Pharmacy Revenue by Department

A column chart showing pharmacy revenue generated by each department.

#### Transaction Distribution by Age Group

A column chart analyzing transaction activity across patient age categories.

#### Cost versus Length of Stay Analysis

A combination chart comparing pharmacy costs against patient length of stay for different diagnoses.

## Data Model

The solution follows a star schema style design where:

Fact tables store transactional healthcare and pharmacy data.

Dimension tables provide descriptive patient and transaction attributes.

Relationships connect visits, patients, and pharmacy transactions to enable integrated reporting and analysis.
